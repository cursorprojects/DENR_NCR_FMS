1. System Overview

The Fleet Management System is designed to manage vehicle maintenance, repairs, and inspections with a comprehensive workflow that ensures quality control and proper documentation.

Key Features:
  • Vehicle Management: Track vehicle information, status, and history
  • Repair Management: Complete repair workflow with pre/post inspections
  • PMS (Preventive Maintenance Service): Scheduled maintenance with inspection requirements
  • Inspection Reports: Pre and post-inspection documentation
  • Granular Permissions: Customizable user access control system
  • Notifications: Real-time alerts and updates
  • Reports: Analytics and reporting capabilities
  • Comprehensive Sample Data: Pre-loaded with realistic test data
Important Notes
  • All repair and PMS processes require proper inspection workflow. You cannot complete repairs or PMS without approved pre and post-inspection reports.
  • Vehicles marked "For Disposal" cannot have new repairs or PMS added. Check vehicle disposal usage percentage on the vehicle list.
  • Always set the acquisition cost when adding vehicles, as it's required for the disposal threshold calculation.

2. Vehicle Management

Adding a New Vehicle:
  1. Navigate to Vehicles in the main menu
  2. Click "Add Vehicle" button
  3. Fill in all required information:
    • Plate Number (unique identifier)
    • Vehicle Type: Select from available options:
      • SEDAN
      • SUV
      • TRUCK
      • PICK UP
      • MOTORCYCLE
      • CLOSE VAN
    • Make, Model, Year
    • Engine and Chassis Numbers
    • Acquisition details and cost (required for disposal threshold calculation)
    • Current mileage and status
  4. Click "Save" to create the vehicle
Viewing Vehicle Details:
  1. Go to Vehicles list
  2. Click on any vehicle plate number
  3. View comprehensive vehicle information including:
    • Basic vehicle details
    • Repair history
    • PMS history
    • Inspection reports
    • Status changes
Changing Vehicle Status:
  1. From vehicle detail page, click "Change Status" button
  2. Select new status and provide reason
  3. Available statuses:
    • Serviceable: Vehicle is operational and ready for use
    • Under Repair: Vehicle is currently being repaired
    • Unserviceable: Vehicle is not operational
    • For Disposal: Vehicle has exceeded disposal threshold (automatic or manual)
  4. Click "Update Status"
  5. Status change will be logged with timestamp and user
Vehicle Disposal & Overuse Condition:
Important

The system automatically monitors vehicle repair costs and marks vehicles for disposal when they reach the overuse threshold.

Disposal Threshold Formula:

  • x = Acquisition Cost ÷ 2
  • Threshold = 30% of x (or 15% of acquisition cost)
  • Vehicle is marked "For Disposal" when total repair costs ≥ threshold

Disposal Status Effects:

  • Vehicles marked "For Disposal" cannot have new repairs or PMS added
  • The system will automatically mark vehicles when threshold is exceeded
  • Vehicles can be manually marked for disposal by changing status

Viewing Disposal Usage:

  • Vehicle list shows a "Disposal Usage" column with percentage indicator
  • Progress bar shows:
    • Green: Below 50% of threshold
    • Blue: 50-79% of threshold
    • Yellow: 80-99% of threshold (Near Threshold)
    • Red: 100% or above (Exceeded Threshold)
  • Total repair costs and threshold amount are displayed
  • Vehicles exceeding threshold are highlighted in red on the vehicle list

3. Repair Process

Important
  • Repairs require a complete inspection workflow. You cannot create a repair without an approved pre-inspection report, and you cannot mark a repair as completed without an approved post-inspection report.
  • Repairs cannot be created for vehicles marked "For Disposal". Check vehicle disposal status before creating repairs.
  • Use the "Add Post-Inspection" button on the repair detail page for faster workflow - the form will be automatically pre-filled.

Step-by-Step Repair Process:
1Create Pre-Inspection Report
  1. Go to Inspections → Pre-Inspections
  2. Click "Add Pre-Inspection"
  3. Select vehicle and report type "Repair"
  4. Fill out comprehensive inspection checklist:
    • Engine, transmission, brakes condition
    • Suspension, electrical, body condition
    • Tires and lights condition
    • Current mileage and fuel level
    • Issues found and safety concerns
    • Recommended actions
  5. Click "Save"
2Approve Pre-Inspection
  1. Go to the created pre-inspection report
  2. Click "Approve" button
  3. Add approval notes (optional)
  4. Click "Approve Report"
  5. Report status changes to "Approved"
3Create Repair Record
  1. Go to Repairs → "Add Repair Record"
  2. Select vehicle and approved pre-inspection report
  3. Fill in repair details:
    • Date of repair
    • Description of work
    • Cost and labor cost
    • Repair shop and technician
    • Status: "Ongoing"
  4. Add parts if needed
  5. Click "Save"
4Create Post-Inspection
  1. After repair work is completed, go to the repair detail page
  2. Option 1 (Recommended): Click the "Add Post-Inspection" button on the repair detail page
    • Form will be automatically pre-filled with vehicle, report type, and pre-inspection
  3. Option 2: Go to Inspections → Post-Inspections → "Add Post-Inspection"
    • Manually select vehicle and report type "Repair"
    • Link to the pre-inspection report
  4. Fill out post-inspection checklist:
    • Work completion assessment
    • Quality, timeliness, cleanliness ratings
    • Post-work condition assessment
    • Test drive results
    • Remaining issues and recommendations
  5. Click "Save"
5Approve Post-Inspection
  1. Go to the created post-inspection report
  2. Click "Approve" button
  3. Add approval notes
  4. Click "Approve Report"
6Complete Repair
  1. Go to repair detail page
  2. Click "Complete Repair" button (appears only after post-inspection approval)
  3. Review completion details
  4. Click "Complete Repair"
  5. Repair status changes to "Completed"
  6. Vehicle status automatically updates to "Serviceable"
What Happens When You Complete a Repair?
  • Repair status changes to "Completed"
  • Vehicle status updates to "Serviceable" (if no other ongoing repairs)
  • Activity log entry is created
  • Notifications are sent to relevant users
  • Status cannot be changed back from "Completed"

5. Inspection Workflow

Pre-Inspection Report Details:
Required Information:
  • Vehicle Selection: Choose the vehicle to inspect
  • Report Type: Repair or PMS
  • Inspected By: Person conducting inspection
  • Condition Assessment: All vehicle systems
  • Mileage: Current odometer reading
  • Fuel Level: Current fuel status
Condition Ratings:
  • Excellent: Perfect condition
  • Good: Minor wear, fully functional
  • Fair: Some wear, needs attention
  • Poor: Significant issues, needs repair
  • Critical: Safety hazard, immediate attention
Post-Inspection Report Details:
Work Assessment:
  • Work Completed Satisfactorily: Yes/No
  • Quality of Work: Excellent to Unsatisfactory
  • Timeliness: How quickly work was done
  • Cleanliness: Work area condition
Post-Work Assessment:
  • Condition Check: All vehicle systems re-evaluated
  • Test Drive: Distance and notes
  • Remaining Issues: Any unresolved problems
  • Future Recommendations: Maintenance advice
Approval Process

Both pre and post-inspection reports must be approved by a supervisor before they can be used in repair or PMS processes. Only approved reports will appear in the dropdown selections.

10. Troubleshooting

Common Issues and Solutions:

Solution: You must first create and approve a pre-inspection report before creating a repair or PMS record. Go to Inspections → Pre-Inspections → Add Pre-Inspection.

Solution: You must create and approve a post-inspection report first. Use the "Complete Repair" or "Complete PMS" button on the detail page, not the edit form.

Solution: Only approved pre-inspection reports appear in dropdowns. Make sure the pre-inspection report is approved by going to the report detail page and clicking "Approve".

Solution: Use the "Change Status" button on the vehicle detail page, not the edit form. Provide a reason for the status change.

Solution: Only users with "Admin Dashboard" permission can access admin features. Contact an admin to grant this permission.

Solution: Only users with "Approve Inspections" permission can approve reports. Contact an admin to grant this permission.

Solution: Menu items are hidden based on user permissions. You need the appropriate "View" permission for each module to see it in the menu.

11. Best Practices

Workflow Best Practices:
  • Always create pre-inspection first: Before any repair or PMS work
  • Be thorough in inspections: Detailed inspections prevent future issues
  • Approve inspections promptly: Don't delay the workflow
  • Use proper status changes: Follow the correct process
  • Document everything: Add detailed notes and descriptions
Data Entry Best Practices:
  • Use consistent naming: Follow naming conventions
  • Enter accurate costs: Include all parts and labor
  • Update mileage regularly: Keep vehicle records current
  • Add detailed descriptions: Helpful for future reference
  • Upload photos when possible: Visual documentation is valuable
System Maintenance:
  • Regular backups: Ensure data is backed up regularly
  • User training: Train users on proper workflows
  • Monitor notifications: Respond to alerts promptly
  • Review reports: Use analytics to improve processes
  • Keep system updated: Apply updates and patches
Permission Management Best Practices:
  • Principle of Least Privilege: Grant only the minimum permissions needed
  • Regular Permission Reviews: Periodically audit user permissions
  • Role-Based Templates: Create permission templates for common roles
  • Document Permission Changes: Keep track of permission modifications
  • Test Permission Changes: Verify permissions work as expected
  • User Onboarding: Set up permissions during user creation
  • Offboarding Process: Remove or disable permissions when users leave
